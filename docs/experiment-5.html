Text and Sets

Working with strings, and the collection built for comparison

Most real data arrives as text. Knowing how to slice it is a superpower.
The big idea How it works The tasks Pitfalls

The big idea

Strings are the format the world hands you data in: names, addresses, log lines, CSV rows, web pages. Python treats a string as an ordered sequence of characters, which means everything you know about positions and slicing applies directly.

Two properties matter most. First, strings are indexed — s[0] is the first character, and negative indices count from the end, so s[-1] is the last one without needing to know the length. Second, strings are immutable: you cannot change a character in place. Every "modification" — .upper(), .replace(), .strip() — returns a brand new string and leaves the original untouched. Forgetting this produces code that looks right and does nothing.

Sets come from a different direction entirely. A set is an unordered collection with no duplicates, and its superpower is comparison. Asking "which items appear in both of these lists?" with loops takes a nested pass; with sets it is a single &. Removing duplicates from a list is set(items). Checking membership in a large collection is dramatically faster in a set than a list, because sets are built on hashing rather than scanning.

How it works

Slicing

s[start:stop:step] — stop is excluded, any part can be omitted:

s = "PYTHON"
s[0]      # 'P'        first character
s[-1]     # 'N'        last character
s[1:4]    # 'YTH'      positions 1, 2, 3
s[::-1]   # 'NOHTYP'   the whole thing, backwards

That last one — [::-1] — is the standard Python idiom for reversing, and it is why the palindrome check in Experiment 4 is a single line.

String methods worth memorising

Method Does
.upper() / .lower() change case (returns a new string)
.strip() remove leading/trailing whitespace
.split() break into a list on whitespace or a separator
.join(list) the reverse — glue a list into one string
.find(sub) position of a substring, or -1
.count(sub) how many non-overlapping occurrences

That word non-overlapping is why Experiment 5 counts occurrences manually: "ABABAB".count("ABA") returns 1, but there are genuinely two overlapping matches. When overlaps matter, scan with .find() advancing the start by one each time.

Set algebra

Operator Meaning
\| union — everything in either
& intersection — only what is in both
- difference — in the left, not the right
^ symmetric difference — in one but not both

Counter — tallying done for you

collections.Counter takes any sequence and returns how many times each item appeared. It replaces the familiar "make an empty dict, loop, get(key, 0) + 1" pattern with a single line.

Task 4

Task 4 — Counting Substring Occurrences (Overlapping Allowed)

str.count() only counts non-overlapping matches, so a manual scan is used to also catch overlapping ones, moving the search start by just 1 each time.

In [4]:
haystack = "ABABABAB"
needle = "ABA"

hits = 0
pos = 0
while True:
    pos = haystack.find(needle, pos)
    if pos == -1:
        break
    hits += 1
    pos += 1

print(f"'{needle}' occurs {hits} time(s) in '{haystack}'")
Out[4]:
'ABA' occurs 3 time(s) in 'ABABABAB'

Task 5

Task 5 — Case-Insensitive Alphabet Frequency

Uses collections.Counter instead of a manual dictionary for tallying.

In [5]:
from collections import Counter

mixed_case = "aAbBBcCCcAa"
letter_tally = Counter(ch.upper() for ch in mixed_case if ch.isalpha())

for letter in sorted(letter_tally):
    print(f"{letter_tally[letter]}{letter}")
Out[5]:
4A
3B
4C

Task 7

Task 7 — Two Fruit Baskets

Overlap, exclusives, and total unique count between two sets.

In [7]:
basket_one = {"papaya", "banana", "litchi", "guava"}
basket_two = {"banana", "fig", "kiwi", "guava"}

print("In both baskets   :", basket_one & basket_two)
print("Only in basket one:", basket_one - basket_two)
print("Total unique fruit:", len(basket_one | basket_two))

Task 8 — Full Set Algebra on Colors

In [8]:
palette_one = {"crimson", "gold", "indigo", "teal"}
palette_two = {"lavender", "teal", "maroon"}

print("Union              :", palette_one | palette_two)
print("Intersection       :", palette_one & palette_two)
print("Only in palette_one:", palette_one - palette_two)
print("Only in palette_two:", palette_two - palette_one)

Common pitfalls

Expecting string methods to modify in placetext.upper() returns a new string. You must assign it: text = text.upper().
.count() and overlapsIt counts non-overlapping matches only. Scan manually when overlaps matter.
Assuming sets keep orderSets are unordered. If order matters, use a list — or sorted(my_set).
Trying to put a list in a setSet members must be hashable. Lists aren't; tuples are.
