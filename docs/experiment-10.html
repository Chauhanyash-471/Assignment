NumPy, Pandas and Matplotlib — the data science stack

A table of numbers hides its story. A chart tells it in a second.

The big idea

This final chapter is where everything converges — and where Python stops being a general-purpose language and becomes the tool of choice for data work.

Three libraries do the heavy lifting, and each solves a distinct problem.

NumPy provides the ndarray: a grid of numbers, all the same type, stored in one continuous block of memory. That layout allows vectorisation — writing arr * 2 and having the multiplication happen across every element at once, in compiled C, without a Python loop. For large data this is not a small improvement; it is often a hundred times faster.

Pandas builds on NumPy and adds what real data actually needs: labels. A DataFrame is a table with named columns and an index, capable of holding mixed types, and — crucially — able to represent missing values. Real datasets always have gaps, and NaN gives them a first-class representation rather than a silent zero that quietly corrupts every average you compute.

Matplotlib turns the result into a picture. Different chart types answer different questions: a line chart shows change over time, a bar chart compares categories, a scatter plot reveals relationships between two variables, and a pie chart shows composition. Choosing the wrong one hides the very thing you were trying to show.

How it works

NumPy essentials

grid.sum(axis=1)    # sum ACROSS each row
grid.sum(axis=0)    # sum DOWN each column

The axis argument confuses everyone at first. The rule: axis=0 moves down through rows (giving a per-column result), axis=1 moves across columns (giving a per-row result).

@ performs matrix multiplication, which is not element-wise — each output cell is the dot product of a row and a column. * on two arrays multiplies element by element instead. Two very different operations, one character apart.

Pandas essentials

Call Purpose
df.head(n) first n rows — always your first look
df.info() column types and non-null counts
df.describe() summary statistics for numeric columns
df["col"] select one column (a Series)
df[df["score"] > 10] boolean filtering — keep matching rows
df.fillna(value) replace missing values

Dealing with missing data

There are three honest options, and the right one depends on context:

  1. Drop the rows (dropna()) — safe when few, wasteful when many
  2. Fill with a statistic like the mean (fillna(df["x"].mean()")) — keeps the row count, but shrinks the apparent variance
  3. Leave it — many Pandas operations skip NaN sensibly on their own

What you must never do is silently replace missing with 0. A missing exam score is not a score of zero, and treating it as one will drag every average you compute downward.

Choosing a chart

Chart Answers
Line how did this change over time?
Bar how do these categories compare?
Scatter are these two variables related?
Pie what share does each part hold?

plt.subplots(2, 2) returns a figure and a grid of axes, letting you place four charts in one figure — which is exactly how this experiment finishes.

Task 2 — A 3x3 Matrix: Row Sums, Column Sums, Second Max

np.unique returns sorted unique values, so the second-to-last entry is the second-largest element overall.

In [2]:
grid = np.array([
    [9, 14, 3],
    [21, 6, 17],
    [8, 25, 11],
])

row_totals = grid.sum(axis=1)
col_totals = grid.sum(axis=0)
second_largest = np.unique(grid)[-2]

Task 7

Task 7 — Four Chart Types in One Figure

Line, bar, scatter, and pie, arranged in a 2x2 grid to compare styles side by side.

In [7]:
import matplotlib.pyplot as plt

quarters = ["Q1", "Q2", "Q3", "Q4", "Q5", "Q6"]
revenue = [210, 225, 240, 260, 255, 280]
costs = [150, 160, 158, 170, 175, 180]

spend_categories = ["Salaries", "Marketing", "Ops", "Misc"]
spend_share = [45, 20, 25, 10]

fig, panels = plt.subplots(2, 2, figsize=(12, 8))

panels[0, 0].plot(quarters, revenue, marker="s", color="darkgreen")
panels[0, 0].set_title("Line: Revenue Over Time")
panels[0, 0].set_ylabel("Revenue")
panels[0, 0].grid(True, linestyle=":", alpha=0.6)

panels[0, 1].bar(quarters, costs, color="steelblue")
panels[0, 1].set_title("Bar: Costs Over Time")
panels[0, 1].set_ylabel("Costs")

panels[1, 0].scatter(revenue, costs, color="firebrick")
panels[1, 0].set_title("Scatter: Revenue vs Costs")
panels[1, 0].set_xlabel("Revenue")
panels[1, 0].set_ylabel("Costs")

panels[1, 1].pie(spend_share, labels=spend_categories, autopct="%1.1f%%", startangle=120)
panels[1, 1].set_title("Pie: Spending Breakdown")

plt.tight_layout()
plt.show()

Common pitfalls

Confusing axis=0 and axis=1axis=0 collapses rows (per-column result); axis=1 collapses columns (per-row result).
* vs @* multiplies element-wise. @ does true matrix multiplication. They give different answers.
Filling missing values with 0A missing score isn't a zero score. Use the mean, or drop the row, and say which you did.
Pie charts with many slicesBeyond about five categories a pie becomes unreadable. Use a bar chart.
